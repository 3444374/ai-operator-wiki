# 知识库同步规则

> 项目 AGENTS.md §11 的对应版本。项目仓库是知识的唯一来源，知识库是编译后的查询界面。

## 工作流

```
项目中有知识变更 → cd ../ai-operator-wiki && ./sync-wiki.sh → 打开 Obsidian → Ingest
```

## 自动同步（通配符，无需手动配置）

- `research/*.md` → `raw/references/`
- `reading_notes/*.md` → `raw/papers/`
- `reference/*.pdf` → `raw/papers/`
- `experiments/plans/*.md` → `experiments/plans/`

## 手动同步（新增文件/目录时加一行 cp）

- `opening/literature/` 顶层 `.md` → `raw/analysis/` 或 `raw/inventory/`
- 新建的知识目录（如 `research/算子代价估计/`）→ 在 `sync-wiki.sh` 加行

## 不同步

代码、CSV、AGENTS.md、README.md、沟通记录

## 新增知识目录的步骤

1. 在 `sync-wiki.sh` 对应区块加 `cp "$PROJECT/新路径/"*.md raw/目标/`
2. 在项目 `PROJECT_LOG.md` 记录
3. `git commit` 更新后的脚本
