## 定义
数据预取是 [[entities/gaussml|GaussML]] 中配合 [[concepts/simd-加速|SIMD 加速]] 使用的一种底层优化技术，通过在数据被实际使用之前将其预先加载到 CPU 缓存中，以减少缓存未命中导致的处理器停顿。在批量 ML 训练的过程中，数据预取能够将计算与访存进行重叠流水化，尤其在大规模特征矩阵的处理中效果显著。该技术直接弥补了传统 [[concepts/ml-as-udf|ML 用户定义函数]] 方案中因数据管理脱离数据库内核而无法实现的细粒度缓存优化。

数据预取通过预测 ML 算子将要访问的内存区域来主动将数据加载到缓存中，从而避免计算单元因等待数据而产生的停顿，与 SIMD 加速形成合力，共同提高端到端的训练性能。它明确面向传统 CPU 执行环境（非 GPU），是针对 CPU 微架构的重要优化手段[^1]。

## 关键特征
- 在数据被计算单元使用前，主动将其从主存搬移至 CPU 高速缓存
- 与 SIMD 向量化执行紧密配合，最大化现代 CPU 微架构的吞吐能力
- 实现计算指令与内存加载指令的流水线重叠，隐藏访存延迟
- 显著减少处理器因缓存未命中（cache miss）而产生的停滞周期，从而有效“减少缓存 miss”[^1]
- 针对批量训练中海量特征矩阵的规律性访问模式进行针对性优化
- 弥补 UDF 方案无法接触内核内存管理的短板，实现数据库内算子级缓存控制
- 是 [[entities/gaussml|GaussML]] 相对于 [[entities/apache-madlib|Apache MADlib]] 获得 2‑6 倍性能优势的底层支柱之一
- 该优化技术明确面向传统 CPU 执行环境，与 GPU 加速方案形成清晰区分与互补
- 对于传统机器学习训练，SIMD 与数据预取是主要的性能瓶颈；但在大语言模型（LLM）推理中，瓶颈转移至 GPU 内存带宽和 KV 缓存，削弱了该技术的直接适用性[^2]

## 应用
- 数据库内大批量 ML 训练任务，如线性模型、逻辑回归等迭代式算法的特征矩阵遍历
- 需在单个查询内对大规模表格进行多次全表或局部扫描的机器学习算子执行场景
- 任何需要充分发挥现代 CPU 微架构特性以提升数值计算效率的数据库内分析负载
